# Product Terminology

Consistent naming across the app, docs, and marketing.

| Term | Use | Don't Use |
|------|-----|-----------|
| Sign in | For authentication | Log in, Login |
| Sign out | For de-authentication | Log out, Logout |
| Dashboard | Main authenticated view | Home page (when logged in) |
| Module | Backend feature unit | Plugin, Extension |
| Component | React UI element | Widget |
| Post | Content item | Article (unless blog) |
| User | Person with account | Customer (unless billing) |
| Admin | Administrative role | Super user |
| API key | Authentication credential | Token (for API keys) |
| Workspace | Organization/team scope | Tenant |

## Capitalization

- Product name: **Vibe on Rails** (not "vibe on rails" or "VIBE ON RAILS")
- CLI command: `vibe` (lowercase in code context)
- Package names: `@vibeonrails/core` (always lowercase)
