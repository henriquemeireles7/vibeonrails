## Why VibeonRails?

<CardGrid stagger>
  <Card title="End-to-End Type Safety" icon="approve-check">

  </Card>
  <Card title="Convention over Configuration" icon="setting">

    Built for AI-assisted development. Clear patterns, zero ambiguity, SKILL.md documentation in every package.
  </Card>
  <Card title="Full-Stack" icon="laptop">

## The Stack

| Layer | Technology | Why |
|-------|-----------|-----|
| **HTTP Server** | Hono | Fast, edge-ready, middleware support |
| **API Layer** | tRPC | End-to-end type safety, no code generation |
| **Database** | Drizzle ORM | Type-safe SQL, migrations, zero overhead |
| **Validation** | Zod | Schema validation, type inference |
| **Auth** | jose + Argon2 | JWT tokens, secure password hashing |
| **Queue** | BullMQ | Redis-based background jobs |
| **Email** | Resend | Transactional email delivery |
| **Frontend** | React | Component library, hooks, CSS system |
| **Testing** | Vitest | Fast, TypeScript-native test runner |
| **CLI** | Custom | Project scaffolding, code generators |

## Quick Example

```typescript
// Create a type-safe API endpoint in 30 seconds
import { createServer, createRouter, publicProcedure } from '@vibeonrails/core/api';
import { z } from 'zod';

const appRouter = createRouter({
  hello: publicProcedure
    .input(z.object({ name: z.string() }))
    .query(({ input }) => {
      return { message: `Hello, ${input.name}!` };
    }),
});

const server = createServer({ router: appRouter });
export default server;
```
